## RED And Root Cause

Original RED evidence is recorded in `.hermes-loop/work/phase-11/red_result.md`.

The review repeat narrowed on two blockers:

- LongMemEval `3b6f954b`: expected evidence was retrieved, selected, rendered, present in answer evidence, and cited, but the answer omitted the country qualifier "Australia".
- LoCoMo `conv-26_qa_028`: expected evidence was retrieved, selected, rendered, and present in answer evidence, but the answer refused and cited no expected source.

Root cause found in the real answer path: Phase 11 serialized full internal `AnswerEvidence.metadata` into the LLM prompt. That metadata contains ranking features, packet ids, neighbor traces, and other diagnostics. Reports need it, but the answerer should receive compact answer evidence.

Added focused regression tests:

- `test_public_answerer_omits_internal_diagnostics_metadata_from_prompt`
- `test_public_answerer_preserves_explicit_location_qualifier_without_metadata_noise`

Fix:

- `PublicAnswerer._answer_evidence_context()` now keeps the answer prompt compact: `id`, `component`, `source_ids`, `session_id`, `date`, `rendered_index`, and `text`.
- `PublicBenchmarkResult.answer_evidence` still preserves full diagnostic metadata in reports.
