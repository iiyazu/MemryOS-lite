# phase: phase-1

## Letta Gap Matrix

Active goal:

> Improve MemoryOS Lite v3 into a benchmark-usable Letta-style agent memory system for LongMemEval and LoCoMo, without demo-only phase completion, without hiding case-level regressions, and without enabling the v3 kernel by default.

This execute-lane matrix consumed `work/phase-1/context_bundle.md`, `work/phase-1/god_dispatch.json`, `work/phase-1/research.md`, and `work/phase-0/baseline_case_matrix.md`. It treats Letta as a reference design only. It does not propose a Letta runtime dependency, does not change code, and does not promote `MEMORYOS_AGENT_KERNEL=v1` to default.

Baseline interpretation:

- LongMemEval Phase 0 smoke separates evidence-hit answer failures (`e47becba`, `118b2229`, `51a45a95`) from retrieval miss (`58bf7951`) and pass (`1e043500`).
- LoCoMo Phase 0 smoke separates evidence-hit answer failure (`conv-26_qa_001`) from retrieval misses (`conv-26_qa_002`, `conv-26_qa_003`, `conv-26_qa_004`, `conv-26_qa_005`).
- `source_hit` is a final projection/source-overlap signal, not pure evidence localization. Future work must continue to inspect `episode_source_hit_at_10`, `planned_evidence_source_hit_at_5`, v3 context diagnostics, answer status, and case-level movement separately.
- Constraints preserved: `MEMORYOS_MEMORY_ARCH=v1` remains the fallback path, v3 remains the intended default architecture to verify through the real CLI/public benchmark path, and `MEMORYOS_AGENT_KERNEL=v1` remains opt-in.

## Priority Summary

P0 contracts for the next implementation phases:

1. Enforce or diagnosably resolve archive attachment/scope in v3 archival retrieval, anchored to LoCoMo retrieval misses.
2. Define a source-vs-agent passage role invariant, anchored to source attribution and LoCoMo retrieval diagnostics.
3. Require passage/message-level citations to survive into final answer artifacts, anchored to LongMemEval and `conv-26_qa_001` evidence-hit answer failures.
4. Add an answer citation/unsupported contract that keeps retrieval misses separate from evidence-hit answer failures.
5. Verify default v3 routing through the real service/public benchmark path while preserving explicit v1 fallback.
6. Preserve public benchmark case-level diagnostics and label `source_hit` conservatively as final projection/source overlap.

P1 contracts:

1. Add explicit core-memory write/read-only or write-policy semantics before expanding kernel mutation.
2. Extend layer-level budget diagnostics with rendered-component accounting.
3. Keep kernel/tool result work opt-in, source-backed, and traceable; do not treat trace presence as answer-quality evidence.
